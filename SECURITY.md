# Security Policy

## Supported Versions

This project is currently in early development.

| Version | Supported |
|---------|-----------|
| v0.1.x  | Yes       |

## Reporting a Vulnerability

If you discover a security issue, please do not open a public issue.

Instead, contact the maintainer privately:

**Email:** 278092697+pingerk03-cell@users.noreply.github.com

Please include:

- a clear description of the issue;
- steps to reproduce it;
- affected files or sections;
- possible impact;
- suggested fix, if available.

## Secret Handling

Never commit:

- API keys;
- access tokens;
- passwords;
- private keys;
- `.env` files;
- personal documents;
- client data.

Use `.env.example` for safe examples.

## Security Principles

- Public repositories must not contain private data.
- Client materials must stay private.
- Secrets must never be stored in Git.
- Security issues should be handled responsibly.
- If unsure whether something is sensitive, keep it private.
